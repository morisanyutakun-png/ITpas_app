# 法務メモ — 「ITパス理解ノート」を安全に運営するために

> このドキュメントは公式な法律意見ではありません。実際にビジネスとして公開・収益化する前に、必ずIT/著作権に詳しい弁護士・司法書士に相談してください。

## 1. 何が問題になり得るか

ITパスポート試験の **問題文・選択肢・公式解答** は、独立行政法人 情報処理推進機構 (IPA) の著作物です。著作権法による保護対象であり、無断で複製・公衆送信すると著作権侵害になり得ます。

「過去問が公開されている」 ＝ 「自由に使える」ではありません。
公開とパブリックドメインは違います。

## 2. 適法に使う3つの道筋

### A. 引用 (著作権法第32条)

引用が成立するための要件:

1. **公表済みの著作物であること** ✓ IPA公式サイトで公表済
2. **引用部分が明瞭に区別されていること** — 鍵カッコ・引用ブロック等
3. **出典が明記されていること** — 出典・年度・問番号
4. **必然性 (引用の必要性)** — 自分の主張のために必要
5. **主従関係** — **自分の創作物が「主」、引用部分が「従」** であること
6. **公正な慣行** — 学術・批評・研究などの正当な目的

**最も難しいのが (5) 主従関係。** 過去問を400問載せて、解説が各問1〜2行、というアプリは「引用の主従関係を満たさない」と判断されやすい。

「主」を強くするには:
- 各問題に **長く深い解説・分析** を付ける（whyAttractive、keyInsight、commonMistakeFlow、関連論点、補助資料）
- 引用問題数を絞り、1問1問を深く扱う（量より質）
- アプリの本体機能（誤解パターン分析、論点マップ、ヒートマップ等）に学習価値の中心がある

本サイトは構造的にこの方向で設計されています。

### B. IPAに利用許諾を得る (推奨)

最も安全なのは **IPAに直接利用許諾を申請** すること。
IPAは過去に教育機関や教材出版社に利用許諾を与えた実績があります。

申請窓口:
- IPA 国家試験問い合わせ窓口: https://www.ipa.go.jp/shiken/inquiry.html
- 利用目的・サイトURL・想定規模・収益モデルを書いて問い合わせる
- 商用利用の場合は別途有償ライセンスの可能性もある

許諾が得られれば「ライセンス供与済み」と明記でき、量的制限の議論を回避できる。

### C. 完全オリジナル問題で運用

過去問を一切使わず、出題範囲（シラバス）に基づくオリジナル問題のみで運用する。
著作権リスクはゼロだが、学習者にとっての「過去問演習」価値は下がる。

## 3. このサイトの現在のスタンス

| 区分 | 対応 |
|---|---|
| **過去問引用 (`originType: "ipa_actual"`)** | 各問題に長文解説・誤答理由分析・関連論点・補助資料を付与し、**引用の主従関係**を満たすよう設計。出典は問題ごとに明記。 |
| **オリジナル問題 (`originType: "ipa_inspired"`)** | シラバス頻出論点を元に独自に作成。出典問題と区別。 |
| **解説・図解・誤解パターン辞書・UI** | 100% 当サイト独自の創作物。 |

## 4. 段階別の運用推奨

### Phase 0 — 個人学習・限定公開 (現在)
- 過去問引用 OK（数十問規模）
- アクセス制限 or 限定公開
- 商用化なし、広告なし

### Phase 1 — 一般公開・無償運営
- 必ず: 充実した解説、出典明記、`/legal` ページ、削除依頼窓口
- 推奨: IPAへ「教育目的での利用」を **事前通知** （許諾が得られれば最強）
- 過去問問題数は **解説の充実度に比例させる**（解説が薄ければ問題数を絞る）

### Phase 2 — 収益化 (広告/有料プラン)
- **必須**: IPAから商用利用の許諾を取得
- もしくは: 過去問は完全に外し、オリジナル問題のみで運営
- もしくは: 過去問版とオリジナル版を分離（過去問版は無償）

## 5. 実務TIPS

### やるべき
- ✅ 各問題ページに **出典クレジット** を表示（実装済）
- ✅ `/legal` ページで著作権スタンスを明示（実装済）
- ✅ 削除依頼の窓口を明記（実装済）
- ✅ Open Graph / sitemap / robots.txt を整備（実装済）
- ✅ 引用問題には **必ず** keyInsight + whyAttractive + 関連論点 + 補助資料 を付ける（運用ルール化）
- ⏭️ サイト規模が拡大するタイミングで IPA に正式に問い合わせ
- ⏭️ プライバシーポリシー、利用規約 を整備
- ⏭️ 個人情報保護法対応（学習履歴を匿名IDで取扱い → 既に匿名運用）

### 避けるべき
- ❌ 過去問のPDFをそのまま再配布する（PDFはダウンロード可能にしない）
- ❌ 過去問を「自分が作ったもの」のように見せる
- ❌ 解説抜きの単なる問題バンクとしてリリースする
- ❌ 過去問だけを切り出して有料化する

### グレーゾーン
- ⚠️ 多年度を通して数百問規模の引用 → 主従関係の解釈で攻めるよりIPA許諾推奨
- ⚠️ 広告 (AdSense等) 表示 → 「実質商用」と判断され得る
- ⚠️ サブスク / 有料プラン → ほぼ確実に商用、許諾必須

## 6. もし IPA から問い合わせがあったら

慌てず、誠実に対応:
1. 利用目的を説明（教育・学習補助）
2. 引用の方法を説明（出典明記、解説付与、主従関係の説明）
3. 必要なら該当部分を一時的に非公開化（`/legal` の削除依頼窓口を使ってもらう）
4. 正式な利用許諾の手続きを相談する

事前に弁護士の連絡先を確保しておくと安心。
